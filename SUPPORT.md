# Support

Use the right channel so maintainers can respond efficiently.

## Website Bugs

Open a GitHub issue for:

- Broken links
- Layout or mobile problems
- Missing images
- Incorrect event, team, project, resource, or partner content
- Accessibility problems

Use the issue templates when available.

## Community Questions

For event registration, community membership, certificates, speaker requests, sponsorship, or partnership questions, use the official contact links on the website instead of opening a code issue.

## Security Issues

Do not publish exploit details in a public issue. Follow [SECURITY.md](SECURITY.md).

## Pull Requests

For code or content contributions, follow [CONTRIBUTING.md](CONTRIBUTING.md).
